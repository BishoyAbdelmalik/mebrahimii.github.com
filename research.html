Big Data Workflow Scheduling in the Cloud

DATAVIEW

 Diabetes is a complex disease and there is a huge demand to capture the range of the disease, including creating predictive algorithms for disease progression using interdisciplinary methods of modeling, computation and medicine. The goal of this research project is to develop a novel, fully automated, machine learning tools with the application of supervised and unsupervised machine learning techniques in order to classify and predicate disease progression. The real patient data is from a continuous glucose monitoring (CGM) study.
Today's big data applications like workflows are often run in cloud computing environments to address the challenges of processing and analyzing data of ever increasing in scale, complexity, and rate of acquisition. In such cloud computing environments, an insider from the cloud provider who has the privilege of an administrator can easily access, leak, or manipulate confidential data that is processed or produced by the execution of scientific workflow tasks. Moreover, external attackers can leverage vulnerabilities of operating systems or hypervisors deployed in the cloud infrastructure to access confidential data or provenance information. We propose to investigate different Trusted Execution Environments (TEEs), in particular SGX, for the development of a trusted execution environment for scientific workflows that will run in the cloud.
When running big data workflows in the cloud, the workflow makespan and the resource utilization efficiency highly depends on the initial placement and distribution of the workflow data and tasks across multiple virtual machines in the Cloud.




The heterogeneous nature of the cloud resources makes the decision of selecting resource type for each workflow a challenging problem. There is still room for improvement, since the scheduling problem is an NP-hard problem.




A big data workflow is the computerized modeling and automation of a process consisting of a set of computational tasks and their data dependencies to process and analyze data of ever increasing in scale, complexity, and rate of acquisition.
